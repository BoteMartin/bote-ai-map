2 — Les outils : ce que l'OS sait faire (11 scripts)

Scripts Python
Ce sont les bras de l'OS. Chaque script se connecte a un service externe et fait un job precis. L'equipe n'a pas besoin de les lancer manuellement : les slash commands (section suivante) s'en chargent. Mais voici ce que chacun fait.
klaviyo_diagnostics.py — Le docteur des comptes Klaviyo 1,903 lignes
A quoi ca sert : Ce script analyse un compte Klaviyo de A a Z et produit un diagnostic complet.
  • Health score : note globale du compte basee sur les flows actifs, les segments, la delivrabilite
  • Benchmark : compare les taux d'ouverture, de clic, de conversion aux standards du secteur
  • Gaps : identifie les flows manquants (ex: "pas de winback flow"), les segments non crees, les problemes de Smart Sending
  • Recommandations : genere une liste de recommandations priorisees avec impact estime
Quand on l'utilise : A chaque audit de nouveau client, et en check-up periodique. C'est le moteur derriere /audit-client et /klaviyo-audit.

Exemple concret : "Lance un diagnostic sur le compte Hygee" → en 2 minutes, on a un rapport de 3 pages avec le health score, les 5 flows manquants, et les 3 segments a creer en priorite.
klaviyo_toolkit.py — L'acces direct aux donnees Klaviyo 1,474 lignes
A quoi ca sert : Lit les donnees live de n'importe quel compte Klaviyo client sans ouvrir le dashboard.
  • Flows : lister tous les flows actifs/drafts/manuels, avec leurs stats
  • Campaigns : voir les dernieres campaigns envoyees, taux d'ouverture, revenue
  • Segments : lister tous les segments, leur taille, leur definition
  • Profils et commandes : chercher un profil, voir son historique d'achat
  • Metriques : revenue par flow, par campaign, par segment
Quand on l'utilise : Chaque fois qu'on a besoin de data Klaviyo. Le script gere automatiquement le rate limiting (pas de risque de bloquer l'API) et la pagination (meme si le client a 500 flows).

Multi-client : Gere 11+ comptes. Il suffit de donner le slug du client et le script va chercher la bonne cle API dans le Keychain.
knowledge_search.py — Le moteur de recherche interne BOTE 881 lignes
A quoi ca sert : Cherche dans toute la knowledge base (36 docs, 14,941 lignes) par sens, pas juste par mot-cle.
  • Ex: chercher "comment optimiser les billing reminders" trouve les bons passages meme s'ils parlent de "touchpoint de retention subscription"
  • Peut filtrer par categorie : -c klaviyo pour chercher uniquement dans les docs Klaviyo
  • Gere un budget de tokens pour ne pas surcharger le contexte de Claude
Quand on l'utilise : Automatiquement par les autres scripts et slash commands. Quand Claude a besoin d'une info interne, il interroge ce moteur pour trouver les passages pertinents.

Ce que ca contient : Docs Klaviyo (API, flows, segments), playbooks CRM, transcripts YouTube (Josh Tay, NLE), audits clients, guides d'outils, 79 learnings cross-clients.
asana_toolkit.py — La vue Asana sans ouvrir Asana 788 lignes
A quoi ca sert : Affiche les tasks Asana directement dans l'OS.
  • Par projet : "Montre-moi les tasks du projet Humble+" → liste par section (To Do, In Progress, Done)
  • Tasks en retard : toutes les tasks overdue de l'equipe en une commande
  • Mes tasks : les tasks assignees a une personne
  • Recherche : trouver une task par mot-cle
Quand on l'utilise : Automatiquement quand on charge un client avec /bote hygee. L'OS affiche les tasks du projet en meme temps que le contexte de la marque. Plus besoin de jongler entre Asana et les fichiers.

Equipe mappee : Les 12 membres de BOTE sont configures avec leur GID Asana.
tldv_toolkit.py — L'archive de tous les calls clients 463 lignes
A quoi ca sert : Retrouve et exploite les transcripts de 150+ calls clients enregistres sur tl;dv.
  • Lister les meetings d'un client ou de toute l'agence
  • Afficher un transcript complet ou les highlights seulement
  • Sauvegarder en .md dans le dossier du client pour reference future
  • Construire un index de tous les calls avec metadata
Quand on l'utilise : Quand on charge un client, l'OS affiche les 5 derniers calls. Aussi utilise pour retrouver une decision specifique : "Dans quel call Mattia a parle des promos ?"
figma-learnings.py — L'extraction des regles clients depuis Figma 605 lignes
A quoi ca sert : Lit les commentaires sur les fichiers Figma et les classe automatiquement.
  • Learning : regle permanente du client (ex: "Toujours utiliser le fond vert" chez LMEM)
  • Action : tache concrete a faire (ex: "Changer le CTA en rouge")
  • Preference : preference de style (ex: "Prefere les photos du dernier shoot")
  • Trivial : "OK", "Valide", etc. → ignore
Quand on l'utilise : Via /figma-sync. Genere ou met a jour le fichier client-rules.md de chaque marque. C'est grace a ca que l'OS connait les preferences exactes de chaque client (854 threads, 22,593 commentaires analyses).
youtube-extract.py — La veille automatisee 204 lignes
A quoi ca sert : Telecharge une video YouTube et la transcrit localement via Whisper (IA de transcription).
  • On lui donne une URL YouTube, il produit un transcript propre
  • Le transcript est ensuite analyse pour en extraire les learnings marketing
  • Les meilleurs learnings sont integres dans la knowledge base
Quand on l'utilise : Veille marketing. On a deja extrait 8 videos (Josh Tay, NLE, Blue Sense, Fleek Marketing, Big Growth Show) qui ont produit 79 learnings sur la retention, la subscription, l'attribution, le BFCM, etc.
Autres scripts utilitaires 4 scripts
  • keyvault.py (434 L) : Fait le pont entre tous les scripts et le Keychain Apple. C'est lui qui recupere les cles API de facon securisee.
  • bote_ask.py (273 L) : Pose une question en langage naturel a la knowledge base et obtient une reponse redigee avec sources.
  • vectorize_knowledge.py (237 L) : Re-indexe la base de connaissances quand on ajoute des docs. A lancer apres chaque ajout.
  • figma-comments.py (169 L) : Version simplifiee de figma-learnings : extraction brute des commentaires sans classification.

3 — Les commandes : comment on s'en sert (9 slash commands)

Slash Commands
Les slash commands sont des raccourcis qui lancent des operations complexes en une ligne. On tape la commande dans Claude Code et l'OS fait le reste : il charge les bons fichiers, appelle les bons scripts, et produit le resultat. C'est l'interface entre l'equipe et toute l'infrastructure.
/bote ou /bote {slug} — Charger le contexte complet 66 lignes
C'est la commande de base. Elle charge tout le contexte BOTE pour etre operationnel immediatement.
  • /bote : charge la config generale, les learnings, les scripts disponibles
  • /bote hygee : charge tout le contexte Hygee (identity, client-rules, figma-comments, segmentation, tasks Asana, derniers calls tl;dv)
  • /bote status : dashboard de tous les clients (tasks, projets)
  • /bote team : tasks du jour par membre de l'equipe
En pratique : C'est la premiere commande qu'on tape en debut de journee ou avant de travailler sur un client.
/audit-client {slug} — Audit complet d'un nouveau client 174 lignes
Pipeline automatise en 7 etapes :
  1. Extraire toutes les donnees Klaviyo du client (flows, segments, campaigns)
  2. Lancer 4 agents de recherche en parallele (site web, fondateurs, concurrents, reviews)
  3. Remplir la fiche d'identite de la marque (copyhub/brand/identity.md)
  4. Analyser les donnees Klaviyo (diagnostic, gaps, benchmarks)
  5. Proposer 20 sujets de newsletters + idees de contenu fondateur
  6. Generer un PDF propre (logo BOTE, tableaux, couleurs)
  7. Envoyer sur Telegram
En pratique : On passe d'un prospect a un audit complet livrable en quelques heures au lieu de plusieurs jours.
/review-mining — Extraire la voix du client 287 lignes — le plus gros skill
Analyse des reviews clients en 12 dimensions : Pain Points, Solutions Echouees, Declencheurs d'Achat, Anxietes, Criteres de Choix, Resultats Desires, Benefices Constates, Transformation, Benefices Inattendus, Expressions Emotionnelles, Langage Specifique, Preuve Sociale.
  • Regle critique : 100% des verbatims sont copies mot pour mot, jamais reformules
  • Produit 2 fichiers : un contexte-brand (pour les copywriters) et un rapport-client (factuel, a envoyer au client)
  • Chaque stat est montree avec le calcul visible (ex: "32/156 = 20.5%")
En pratique : On donne un fichier de reviews (Trustpilot, Amazon, Google) et on obtient tout le material pour ecrire du copy qui parle comme les vrais clients.
/youtube-extract {url} — Veille marketing automatisee 172 lignes
6 etapes : telecharger → transcrire (Whisper) → nettoyer → analyser (resume + learnings + verbatims) → sauvegarder dans la KB → mettre a jour les global-learnings.

En pratique : On donne une URL YouTube d'une video marketing (Josh Tay, NLE, etc.) et les meilleures idees sont integrees dans notre base de connaissances automatiquement.
/frenchmush — Supprime KILL
Skill legacy, plus utilise. Sera remplace par /copy {slug} (voir roadmap) : un skill generique qui charge automatiquement le copyhub du client demande, quel qu'il soit. Plus besoin d'un skill par marque pour le copy de base.
/klaviyo-audit, /figma-sync, /miro-board, /miro-cj-map 4 skills
  • /klaviyo-audit {slug} (52 L) : audit Klaviyo uniquement (pas l'audit complet) — flows, gaps, performance vs benchmarks, top 5 recommandations
  • /figma-sync {slug} (61 L) : extrait les commentaires Figma et met a jour client-rules.md + figma-comments.md
  • /miro-board (41 L) : transforme une strategie de campagnes en board Miro visuel (phases, segments, couleurs)
  • /miro-cj-map (61 L) : genere une Customer Journey Map sur Miro avec les flows par canal et leur statut

6 — Roadmap : ce qu'on va construire

La vision
L'OS doit devenir le cockpit central de chaque PM et chaque membre de prod. Pas juste un outil pour Martin. Chaque process repetitif (QA, segments, briefs, reporting) doit etre automatise ou assiste. Chaque call client doit etre indexe et cherchable. Chaque erreur recurrente doit etre catchee avant qu'elle parte en production.
Fondations
Poser les bases pour que tout le reste fonctionne.
Vectoriser 150+ calls tl;dv PRIORITE #1
Le probleme : On a 150+ transcripts de calls clients sur tl;dv, mais ils ne sont pas cherchables. Quand un PM veut savoir "qu'est-ce que Mattia a dit sur les promos au dernier call ?", il doit ouvrir tl;dv et chercher manuellement.

Ce qu'on va faire : Extraire tous les transcripts → les decouper en chunks semantiques → les vectoriser dans ChromaDB (comme on a fait pour la knowledge base). Resultat : une recherche semantique instantanee sur toute l'histoire des calls clients.

Impact : On pourra retrouver n'importe quelle decision, feedback, ou demande client en 2 secondes. Les PM n'oublieront plus jamais un retour client.
Brain MCP (Model Context Protocol) INFRASTRUCTURE
Le probleme : Aujourd'hui Claude Code doit lancer un script pour chercher dans la KB. C'est lent et ca casse le flow.

Ce qu'on va faire : Un serveur MCP qui expose toute la memoire de l'agence (calls indexes, KB, regles clients) comme une source de donnees native. Claude pourra y acceder en temps reel, sans lancer de script. Comme un deuxieme cerveau toujours connecte.

Impact : Recherche instantanee dans les calls, la KB, et les regles clients. Chaque conversation avec l'OS sera enrichie automatiquement du contexte pertinent.
Skill /copy {slug} REMPLACE /frenchmush
Le probleme : Le skill /frenchmush etait fait pour un seul client. On a besoin d'un skill generique qui marche pour tous.

Ce qu'on va faire : Un skill /copy {slug} qui charge automatiquement le copyhub du client (identity, audiences, produits, exemples d'emails) et produit du copy dans le ton exact de la marque. Un seul skill pour 15 clients.

Qui l'utilisera : Les PM pour les briefs, la prod pour les emails et NLs.
Re-vectoriser la knowledge base 30 min
La KB n'a pas ete re-indexee depuis longtemps. Les 79 learnings et les docs recents ne sont pas cherchables. Une commande : python3 vectorize_knowledge.py. Doit etre fait avant tout le reste.
Skills PM
Des outils faits pour les Project Managers. Gagner du temps sur les taches repetitives.
/qa {slug} — QA automatique avant envoi HIGH IMPACT
Le probleme : Avant chaque envoi d'email, le PM doit checker manuellement : liens, segments, Smart Sending, orthographe, CTA, produits, footer, UTMs. C'est long et les erreurs passent.

Ce que le skill fera :
  1. Checker tous les liens dans l'email (broken links, UTMs manquants)
  2. Verifier que le bon segment est cible (pas d'envoi a tout le monde par accident)
  3. Verifier Smart Sending et Quiet Hours
  4. Checker l'orthographe et la coherence avec le copyhub du client
  5. Verifier que les CTA pointent vers les bons produits
  6. Generer un rapport de QA avec OK / WARNING / ERREUR
Qui l'utilisera : Chaque PM avant de valider un envoi. Chaque membre de prod pour s'auto-checker.
/segment-check {slug} — Audit segments en continu HIGH IMPACT
Le probleme : Les segments Klaviyo se degradent avec le temps. Des contacts inactifs restent dans les segments engaged, des segments Apple MPP ne sont pas filtres, des segments se chevauchent.

Ce que le skill fera :
  • Comparer les segments du client vs notre framework (30j/60j/90j/180j, MPP filter, VIP, etc.)
  • Detecter les segments manquants
  • Alerter sur les segments trop larges ou trop etroits
  • Verifier la coherence entre segments et campaigns ciblees
  • Proposer des corrections precises
Qui l'utilisera : Les PM en check-up mensuel, et a chaque onboarding client.
/pm-brief {slug} — Prep de call en 2 min PM TOOL
Le probleme : Avant un call client, le PM doit aller sur Asana, ouvrir Klaviyo, relire les derniers commentaires Figma, retrouver les decisions du dernier call. Ca prend 15-20 min.

Ce que le skill fera : Generer un brief pre-call automatique en 2 min :
  • Tasks Asana en cours et en retard
  • Derniers emails envoyes + performance
  • Derniers commentaires Figma non resolus
  • Resume du dernier call (quand les calls seront vectorises)
  • Points a aborder
/reporting {slug} — Reporting mensuel automatise PM TOOL
Extraire les metriques Klaviyo du mois → comparer aux benchmarks sectoriels → comparer au mois precedent → generer un PDF propre avec les charts, les top/flop campaigns, et les recommandations. Le PM n'a plus qu'a relire et envoyer.
Skills Production
Des outils pour la prod : copy, design, QA, learnings.
/flow-check {slug} — Audit des flows en continu HIGH IMPACT
Le probleme : Les flows Klaviyo vivent : des emails sont desactives, des conditions changent, des flows nouveaux ne sont jamais actives. Sans check regulier, des clients perdent du revenue sans qu'on le sache.

Ce que le skill fera :
  • Lister les flows actifs/drafts/manuels et leur status
  • Detecter les flows critiques manquants (Welcome, Abandoned Cart, Post-Purchase, Browse, Winback)
  • Identifier les flows avec des emails desactives ou des taux anormaux
  • Comparer vs l'architecture lifecycle ideale (Option B)
  • Score de sante des flows + recommandations priorisees
/client-prep {slug} — Onboarding auto PROD TOOL
Quand un nouveau client arrive : creer le dossier, lancer l'extraction Klaviyo, scraper le site, remplir identity.md, lancer le diagnostic, sync les commentaires Figma. Tout en une commande au lieu de 7 etapes manuelles.
/learn {slug} — Extraire les learnings d'un client PROD TOOL
Lire tous les commentaires Figma + calls du client, extraire les regles et preferences, mettre a jour client-rules.md et ajouter les nouveaux learnings dans global-learnings.md. Permet de garder la memoire a jour sans effort.
Slack Bot — L'OS accessible sans terminal EQUIPE
Un bot dans Slack qui permet a toute l'equipe de poser des questions a l'OS sans ouvrir Claude Code. "Quelle est la regle de Cut By Fred pour les CTA ?" → reponse directe depuis la KB. Aussi : notifications automatiques quand un segment se degrade, quand un flow a un taux anormal, quand une task est en retard.
Connexions manquantes a construire
Les liens entre les outils existants qui n'existent pas encore et qui debloqueraient de la valeur.
tl;dv → ChromaDB 150+ calls non indexes. On perd toute la memoire des decisions clients. C'est le lien #1 a construire.
Klaviyo → QA automatique Aujourd'hui on check manuellement. Un lien direct permettrait de checker les liens, segments, Smart Sending avant chaque envoi.
Asana → /pm-brief Les tasks sont la, mais pas exploitees automatiquement dans les briefs de call. Connexion directe a faire.
Figma → client-rules auto-update Aujourd'hui il faut lancer /figma-sync manuellement. A terme : les nouveaux commentaires mettent a jour les regles automatiquement.
ChromaDB → MCP Server La KB et les calls indexes doivent etre accessibles en temps reel via MCP, sans lancer de script.
OS → Slack L'equipe doit pouvoir parler a l'OS depuis Slack, pas uniquement depuis le terminal.

7 — Securite

Pas de mot de passe en clair, nulle part
Toutes les cles API (Klaviyo x11, Asana, tl;dv, Figma, Higgsfield) sont dans le Keychain Apple. Les scripts les recuperent au moment de l'execution. Si quelqu'un accede au code, il ne voit aucun mot de passe.
72 regles de permission
Claude Code ne peut pas faire n'importe quoi : les domaines web accessibles sont restreints aux sites clients, les commandes Bash sont limitees (python3, pdflatex, curl), et les operations destructives (git push force, delete) sont bloquees par defaut.
